# 奇遇剧本创作指南

创作模式：将任意概念种子转化为完整、可游玩的剧本。默认行为是先出草稿——立即生成，精准迭代。

---

## 读懂用户状态

从信号判断用户的状态，不要问元问题。

| 信号 | 状态 | Agent 行为 |
|------|------|----------|
| 简短概念抛出、「直接生成吧」、没有细节 | **快速模式** | Agent 主导——立即生成完整草稿 |
| 部分细节、语气犹豫、「类似于……」 | **协作模式** | 共同构建——带标注假设的草稿，邀请用户反应 |
| 用自然语言预填了字段内容 | **专家模式** | 格式化 + 轻度打磨，不添加额外创作 |

**默认：快速模式。** 一个草稿胜过十个问题。

---

## 先出草稿的循环

1. 用户给出任意概念种子——一个词就够。
2. Agent 立即生成全部五个字段，行内标注假设。
3. 以可读散文（而非 JSON）呈现，附简短的「我假设了 X」说明。
4. 用户反馈 → Agent 只修改变化的字段，不大规模重写。
5. 用户明确确认 → 调用 `create_adventure_campaign`。

**边缘情况**
- 对话中途转向 → 从对话历史综合信息，立即生成新草稿。
- 用户对多个字段不满 → 问一个定向问题，然后重新生成完整草稿。
- 用户提供了完整描述 → 只做格式化和轻度打磨，不添加额外创作。

---

## 五个字段作为一个整体

生成前，参考 `adventure-examples.md` 了解各类型的范例。

### `mission_plot` — 是什么
像开篇段落一样写，而非摘要。世界、情境、钩子。从「已经出了问题」开始——第一句话就是 AI 体验到的第一件事，它决定整个会话的氛围。

强版：*「天机门的内部交接班记录在第十一天戛然而止——不是在一班岗末尾，而是在一句话的中间。」*
弱版：*「你是一名抵达神秘之地的探险者。」*

### `mission_task` — 做什么
玩家的主动使命。足够具体以产生决策，足够开放以允许多种路径。玩家可见。设定目标，不要设定路径。

### `mission_plot_attention` — AI 的运行规则
3–6 条规定 **AI 如何行动** 的规则，而非故事里发生什么。涵盖：基调、NPC 行为契约、信息门控、后果规则、内容护栏。这些规则无条件凌驾于所有其他上下文之上。

**反模式**：不要在这里放世界观或剧情事件——那些属于 `mission_plot`。

强版注意示例：
```
保持心理恐怖基调——用恐惧和暗示，而非露骨内容。
玩家能动性是神圣的：绝不强迫行动或替他们做选择。
所有揭示必须通过调查或 NPC 说服才能获得。
Victoria（绑定角色）有自己的议程；她提供信息但不解决问题。
避免：graphic 暴力、性内容、打破第四面墙。
```

### `name` + `subtitle`
有感召力、简短、略带歧义。名字应暗示类型而非解释它。

### 字段一致性检查
如果 `mission_plot` 是惊悚谍战，`mission_plot_attention` 就应该执行惊悚谍战式的行为规则。字段不匹配会导致游玩漂移。

---

## 创建后

立即调用 `request_adventure_campaign` 确认字段正确存储——尤其是 `mission_plot_attention`，它统治所有后续游玩会话。

首次试玩后，注意 AI 漂移的地方；用 `update_adventure_campaign` 在那些节点收紧 `mission_plot_attention`。

---

## 质量自检清单

- [ ] `mission_plot` 的第一句话是否把读者扔进「已经出了问题」的当下，没有前言？
- [ ] 玩家是否知道目标，但不确切知道路径？
- [ ] `mission_plot_attention` 是否覆盖了最可能破坏沉浸感的 3 个向量？
- [ ] 每个命名 NPC 是否足够独特，台词无法互换？
- [ ] `mission_plot_attention` 是否没有混入世界观内容？
