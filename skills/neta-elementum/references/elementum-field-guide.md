# Elementum Field Reference Manual

Detailed description and filling specifications for all fields in `create_elementum` and `update_elementum`.

---

## Required Fields

### `name`

Element name, max 128 characters.

- Recommend short, distinctive names
- Reference in `make_image` via `/ElementName` after creation
- Example: `RE4 Village`, `Ink Wash Style`, `Katana`, `Battle Stance`

---

### `artifact_uuid` (required for create)

Element representative image artifact UUID from `make_image` response `artifacts[0].uuid`.

- Representative image should accurately reflect the visual concept
- Recommend choosing the clearest, most typical visual effect
- This image also serves as the element's cover image

---

### `prompt` (required for create)

Direct image generation instruction for the model.

**Two key properties:**

1. **Directly usable** — Can be pasted directly into `make_image --prompt` for standalone use
2. **Composable** — Doesn't conflict when combined with character references or description words

**Writing guidelines:**
- Focus on core visual features, avoid irrelevant descriptions
- No specific characters (unless it's a pose element anchored to a character)
- Language can be English

**Examples by type:**

```
# Scene
Resident Evil 4 style European medieval abandoned village, dilapidated stone houses, burning bonfire, thick fog, dead trees, horror atmosphere, realistic style

# Item
Japanese katana, slender blade, purple scabbard, golden handguard, steel edge, metallic reflection, white background

# Style
Ink wash painting style, negative space aesthetics, landscape conception, natural brushstrokes, traditional Chinese painting

# Pose
Female character, looking back stance, flowing long hair, 45-degree side profile, elegant, white background, full body

# Atmosphere
Cyberpunk neon lighting, purple blue pink mixed halos, rainy night reflective ground, tech feel
```

---

## Optional Fields

### `description`

Agent-facing usage guide. **This is the core field for Agents to understand when and how to use this element.**

**Recommended format:**
```
This element represents [concept description]. Use by [method and pairing suggestions]. Reference image shows [description]. [Notes (if any)].
```

**Examples by type:**

```
# Scene
This element represents the iconic abandoned European village from Resident Evil 4, filled with horror atmosphere.
Use with character references (@CharacterName) and night, dark, horror description words, suitable for horror or game-style image generation.
Reference image is a recreation of the original RE4 game village scene.
Note: Environment-focused; if characters needed, separately describe character actions and positions.

# Style
This element represents Japanese ink wash painting style, emphasizing negative space aesthetics and landscape conception.
Can be used with any scene or character, suitable for poetic, classical, Eastern aesthetics.
Reference image is traditional ink wash style test image. Can overlay with multiple scene descriptions.

# Meme
This element represents the classic "Star Wars" meme — Padmé Amidala pointing to the right.
Use by describing character pointing direction and expression to activate the meme atmosphere.
Reference image is an anime recreation of the original meme style. Suitable for lighthearted and humorous scenes.
```

> **Important:** Description quality directly affects how correctly Agents use this element in subsequent creations. Recommend clearly explaining "when to use" and "how to use".

---

### `ref_image_uuid`

Concept anchor reference image artifact UUID for visual style anchoring (optional).

**Suitable scenarios:**
- Have clear reference source (e.g., game screenshot, movie still)
- Need image models to precisely recreate specific visual style
- Element's visual features are difficult to describe with text alone

Reference image sources:
- `artifacts[0].uuid` from images generated by `make_image`
- Image artifacts obtained via `read_collection`

---

### `accessibility`

Visibility setting.

| Value | Meaning |
|-------|---------|
| `PUBLIC` (default) | All users can search and use |
| `PRIVATE` | Only creator can see and use |

---

## update_elementum Specific Fields

### `tcp_uuid` (required)

Element's unique identifier UUID, obtained from `request_character_or_elementum` or `create_elementum` response.

## Field Relationship Summary

```
┌─────────────────────────────────────────────────────┐
│                 Element Token (TCP/Elementum)       │
│                                                     │
│  For image models           For Agents              │
│  ──────────────             ─────────────────       │
│  prompt (gen instruction)     description (guide)   │
│  ref_image_uuid (style anchor)                      │
│                                                     │
│  anchor                                             │
│  ──────                                             │
│  artifact_uuid (representative image, cover)        │
└─────────────────────────────────────────────────────┘
```

## prompt vs description Comparison

| Dimension | prompt | description |
|-----------|--------|-------------|
| **Reader** | Image model | Agent / User |
| **Purpose** | Direct image generation | Explain how to use |
| **Language** | Visual tags, precise | Natural language, explanatory |
| **Example** | `Cyberpunk neon, rainy night, purple halos` | `This element represents cyberpunk night atmosphere, use with city scenes...` |
