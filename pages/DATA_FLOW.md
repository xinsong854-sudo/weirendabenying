# 数据流说明

## 登录

1. 用户输入手机号与验证码。
2. 前端请求本网站后端 `/api/proxy/request-code` 和 `/api/proxy/verify-code`。
3. 后端仅转发到 Talesofai / Neta 登录接口。
4. 后端不保存手机号和验证码。
5. 登录成功后 Token 返回前端。
6. Token 存入用户浏览器本地 `localStorage`。

## Token 使用

- 存储位置：用户浏览器本地 `localStorage`。
- 请求方式：作为 `x-token` 请求头临时发送给后端。
- 后端用途：本次请求身份验证、调用上游 Neta API。
- 后端持久化：不持久化。

## 论坛 / Wiki / 评论

用户主动发布的内容会保存到后端 SQLite 数据库，用于多用户同步展示。

## 身份卡 / CoC 车卡

1. 用户粘贴 Neta 角色短链。
2. 后端通过 Neta 短链解析接口获取角色 UUID。
3. 后端使用用户本次请求携带的 `x-token` 临时读取角色资料。
4. 后端调用 LLM 生成车卡。
5. 生成成功的身份卡保存到后端 `identity_cards` 表，绑定当前用户 UUID。
6. 表中不包含 Token、手机号、IP。

## 私聊

私聊内容保存于后端数据库，用于站内消息功能。若未来需要更高隐私等级，可改为端到端加密或仅本地保存。
